images

lazyloading
          
- подключаем lazyload.js
- инициализируем по определенному классу изображений (например .lazyload)
- картинкам указываем data-original с путями к оригиналам и в src - заглушку
  (лучше кодированую в base64 - https://webdesign-master.ru/services/lazy/).
  Картинкам нужно указать width/height, что бы не прыгали при отрисовке.
responsive image
          
<picture>
  <source media='(min-width: 1024px)' srcset='images/600x600.webp' type='image/webp'>
  <source media='(max-width: 1023px)' srcset='images/image-1x.webp images/image-2x.webp 2x'
          type='image/webp'>
  <img src='images/image-1x.png' alt='Just image'>
</picture>
          
        
  source в зависимости от ширины экрана (аттрибут media) загружает ту или
иную картинку.
  Это удобно, что бы для моб версии не грузить тяжелую картинку или, например,
если нужна другая картинка для моб. Или, например, для fallback'a для webp (хотя
возможно и обычный <img> c srcset для этого сгодится, где src будет fallback'ом
если не поддерживается webp формат из srcset).
  Если браузер не поддерживает MIME-тип из аттрибута type (например webp) или
нет совпадения с media-запросом, то вместо <source> берется следующий <source>
и так далее. Если ни один <source> не подошел - берется <img>

высота картинки в % относительно ширины
Задаем блоку высоту в % относительно ширины с помощью padding-bottom.
И внутри блока позиционируем картинку на всю высоту/ширину блока
